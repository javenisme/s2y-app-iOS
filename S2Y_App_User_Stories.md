## 1. 现有功能 (Spezi 框架)

### 1.1 核心模块

| 模块 | 功能 | 技术栈 |
|------|------|--------|
| **Health Assistant** | 健康助手对话界面 | SwiftUI |
| **Schedule** | 任务与问卷调度 | Spezi Scheduler |
| **Account** | 用户账户管理 | Spezi Account |
| **HealthKit** | Apple 健康数据集成 | HealthKit |
| **LLM** | AI 对话能力 | LLMProvider (Minimax API) |
| **LocalModel** | 本地模型下载管理 | ModelDownloadManager |
| **Bluetooth** | 设备连接 | CoreBluetooth |
| **Onboarding** | 用户引导流程 | Spezi Onboarding |

### 1.2 现有数据流

```
用户输入 → HealthAssistantView 
         → HealthIntelligenceEngine 
         → LLMProvider (Minimax API)
         → Firestore (聊天历史)
```

## 3. 待实现用户故事

### US-006: 离线健康分析

| 属性 | 内容 |
|------|------|
| **ID** | US-006 |
| **标题** | 无网络时获得健康问题分析 |
| **角色** | 注重隐私的用户 / 经常处于离线状态的用户 |
| **场景** | 用户在飞机上或信号不好时，询问健康问题 |
| **优先级** | P0 (核心差异化) |
| **技术依赖** | MLX Swift, LocalLLMService |

**用户价值:**
- 随时随地获得健康指导，不受网络限制
- 敏感健康数据完全保存在本地，安心

**验收标准:**
- [ ] 离线模式下可使用本地 LLM 进行问答
- [ ] 响应时间 < 5秒 (简单查询)
- [ ] 支持流式输出显示

**实现方案:**
```swift
// 用户旅程
1. 用户打开 App (无网络)
2. 用户向助手提问: "我最近总是胸闷，怎么办？"
3. 系统检测到离线，自动路由到本地 LLM
4. 本地 LLM 结合用户本地健康数据生成回答
5. 用户收到回答，无须等待
```

**验收标准:**
- [ ] 包含"心率"、"血压"等关键词的查询自动走本地

- [ ] 用户可手动切换处理方式

**实现方案:**
```
查询: "我最近心率变异很低" 
  → 包含敏感关键词 
  → 路由到本地 LLM 
  → 结合本地 HRV 数据分析
  → 返回个性化建议

查询: "如何预防心脏病"
  → 通用知识 
  → 路由到云端 LLM 
  → 返回权威医学知识
```

### US-008: 健康知识图谱查询

| 属性 | 内容 |
|------|------|
| **ID** | US-008 |
| **标题** | 查看健康数据之间的关联 |
| **角色** | 想要深入了解自己健康状况的用户 |
| **场景** | 用户想了解为什么最近总是疲劳 |
| **优先级** | P1 (重要) |
| **技术依赖** | SwiftData, SwiftGraph, sqlite-vec |

**用户价值:**
- 了解症状背后的可能原因
- 发现被忽视的健康关联

**验收标准:**
- [ ] 显示症状与生活习惯的关联图
- [ ] 支持点击查看详情
- [ ] 支持时间范围筛选

**实现方案:**
```
用户: "为什么我最近总是疲劳？"
系统: 
  1. 查询本地知识图谱
  2. 发现关联: 疲劳 → 睡眠质量 ↓ → 深度睡眠不足
  3. 发现关联: 睡眠 → 睡前屏幕时间 ↑
  4. 返回分析: "你的疲劳可能与深度睡眠不足有关..."
```

## 4. 用户旅程地图

### 4.1 核心用户旅程

```
┌─────────────────────────────────────────────────────────────────┐
│                      新用户首次使用                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  下载 App → 注册账户 → 引导流程 → 授权 HealthKit                │
│       ↓                                                          │
│  欢迎页面 → 功能介绍 → 完成设置                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                      日常健康查询                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  打开 App → Health Assistant → 输入问题                          │
│       ↓                    ↓                                     │
│   首页展示    ←  AI 回答 (本地/云端)                             │
│       ↓                    ↓                                     │
│  查看健康数据 ←  知识图谱关联                                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                      深度健康分析 (Phase 2)                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  进入仪表盘 → 选择分析类型 → 执行分析                            │
│       ↓                    ↓                                     │
│   查看趋势    ←  生成报告 (RAG)                                  │
│       ↓                    ↓                                     │
│  导出/分享 ←  异常预警                                           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
