## 1. 竞品分析总结

### 1.1 主要竞品

| 应用 | 核心能力 | 隐私/离线 | 设备集成 | AI 能力 |
|------|----------|-----------|----------|---------|
| **August AI** | 健康伴侣 | ✅ HIPAA/GDPR, 本地处理 | - | 对话式 AI |
| **Ada Health** | 症状评估 | ❌ 云端 | - | 症状检查器 |
| **Fitbit** | 健康监测 | ⚠️ 可选 | ✅ 可穿戴 | AI Insights |
| **Woebot** | 心理健康 | ✅ 隐私优先 | - | CBT 对话 |
| **Apple Health** | 健康聚合 | ✅ 苹果生态 | ✅ 设备生态 | 基础 |

### 1.2 市场差距

- **离线 AI**: 大多数竞品依赖云端，本地 LLM 是差异化点
- **复杂分析**: 现有 app 多做简单症状检查，缺乏深度健康问题分析
- **知识图谱**: 几乎没有 app 用知识图谱做个人健康数据的关联分析

### 2.2 Orchestrator 工作流程

```
用户输入
    │
    ▼
┌─────────────────┐
│ IntentClassifier│ ───→ 分类意图 (健康查询 / 设备控制 / 数据分析 / 任务执行)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   TaskPlanner   │ ───→ 规划任务链 (分解为子任务)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ AgentSelector   │ ───→ 选择执行 Agent
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
 Local    Cloud
 Agent    Agent
    │         │
    └────┬────┘
         ▼
┌─────────────────┐
│  ResultMerger   │ ───→ 合并结果，生成最终响应
└─────────────────┘
```

## 3. 功能模块

### 3.1 核心功能 (MVP)

| 功能 | 描述 | Agent 类型 |
|------|------|------------|
| **健康问答** | 基于症状的 AI 健康分析 | Cloud (s2y-omer) |
| **离线健康助手** | 本地 LLM 处理简单查询 | Local |
| **健康数据聚合** | 整合 HealthKit + 设备数据 | - |
| **知识图谱查询** | 用 cognee 分析健康数据关联 | Knowledge |
| **设备控制** | 连接 taVNS 等设备 | Local |

### 3.2 高级功能 (Phase 2+)

| 功能 | 描述 |
|------|------|
| **复杂健康分析** | 多维度健康问题深度分析 |
| **预测健康** | 基于历史数据的健康趋势预测 |
| **个性化方案** | 基于个人数据的健康建议 |
| **隐私报告** | 生成可导出的健康报告 |

## 4. 技术实现

### 4.1 本地 LLM 集成

```swift
// LocalHealthModelManager 扩展
class LocalLLMOrchestrator {
    // 模型选择策略
    enum ModelType {
        case local      // 本地小模型 (Phi-3 Mini 等)
        case hybrid     // 本地 + 云端混合
        case cloud      // 纯云端 (s2y-omer)
    }
    
    // 隐私级别决定走哪个模型
    func route(query: String, privacyLevel: PrivacyLevel) -> ModelType
}
```

### 4.2 s2y-omer 集成

- **方式**: 通过 API 调用 s2y-omer 服务
- **能力**: 
  - 多模型切换 (Claude, Gemini, GPT)
  - 工具调用 (天气、文档、知识库)
  - Artifacts (代码、图表)

### 4.3 s2y-cognee 集成

- **方式**: 本地 Python 服务 (cognee-lite)
- **能力**:
  - 向量存储 (Chroma)
  - 知识图谱 (NetworkX)
  - 文档摄取 (PDF, TXT, JSON)

## 5. 安全与隐私

### 5.1 数据分级

| 级别 | 数据类型 | 处理方式 |
|------|----------|----------|
| **敏感** | 医疗记录、症状 | 100% 本地处理 |
| **内部** | 健康指标、运动 | 本地 + 可选云端 |
| **公开** | 天气、新闻 | 云端处理 |

### 5.2 加密

- **传输**: TLS 1.3
- **存储**: AES-256 (Keychain)
- **本地**: Data Protection API
